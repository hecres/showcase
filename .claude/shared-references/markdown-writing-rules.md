# Markdown記述ルール

Markdownファイルを出力・編集するスキルが共通で参照する記述ルール。

---

## 改行

本文中で改行を意図する箇所には `<br/>` を使用する。<br/>
trailing spaces（行末の半角スペース2つ）は使用しない。視認しづらくエディタの自動トリムで消失するリスクがあるため。

---

## Mermaid図

### ノードラベル内の改行

ノードラベル内で改行する場合は `<br>` を使用する。`\n` はMermaidで改行として認識されない。

```mermaid
%% OK
A["行動カード<br>ActCard"]

%% NG — プレビューで改行されない
B["行動カード\nActCard"]
```

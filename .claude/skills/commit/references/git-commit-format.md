# コミットメッセージフォーマット

[Conventional Commits](https://www.conventionalcommits.org/) に基づくコミットメッセージの記述ルールを定める。

---

## フォーマット

```
<type>: <description>

[body]

[footer]
```

### 1行目（必須）

- `<type>: <description>` の形式で記述する
- `<description>` は変更内容を端的に記述する（日本語可）

### 本文（任意）

- 1行目と本文の間に空行を1行入れる
- 句点（。）で改行する
- 1行目だけで変更内容が明確なシンプルな変更では省略してよい

#### 本文の構成

本文は以下の2パートで構成する。

```
<what: 何をどう変えたかの詳細説明>

<why: なぜその変更を行ったか>
```

- **what**（詳細説明）: 1行目の description では伝えきれない変更の具体的内容を記述する
- **why**（理由）: 変更の動機・背景・意図を記述する
- what と why の間に空行を1行入れる
- いずれか一方のみで十分な場合は、もう一方を省略してよい

### フッター（任意）

- 本文とフッターの間に空行を1行入れる
- `Co-Authored-By:` 等のメタ情報を記述する
- 破壊的変更がある場合は `BREAKING CHANGE: <説明>` を記述する

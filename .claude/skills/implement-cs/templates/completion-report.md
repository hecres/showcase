# 完了報告テンプレート

セルフレビュー完了後、以下のフォーマットで報告書を作成しユーザーに提示する。

```
---

【実装完了報告(implement-cs)】

[依頼内容: 概要]
（依頼内容の1行要約）

[依頼内容: 原文]
（ユーザーが会話で入力した依頼の原文。スキルに渡された ARGUMENTS ではなく、ユーザー自身が入力したプロンプトを転記する）

[対応事項]
- （調査・分析・実装・編集の各事項を箇条書きで列挙）

[技術的注記]
（以下のいずれかに該当する場合のみ記載。該当なしの場合はセクションごと省略）
- 設計上の回避策: rules/アーキテクチャ制約により教科書的設計を採れず、代替手段を選択した箇所
- 潜在リスク: 発生確率は低いが特定パターンで問題が起こりうる箇所
- 意図的な複雑性: セキュリティ・パフォーマンス等の理由で敢えて複雑な実装を選択した箇所
- その他: 将来の変更時に注意が必要な依存関係や前提条件

---
```

## 記載ルール

- 依頼内容: 概要を1行で記載し、原文をそのまま転記する
- 対応事項: 調査→分析→実装の流れに沿って、実際に行ったことを漏れなく列挙する
- 技術的注記: 該当する場合のみ記載する。単純な実装では省略してよい

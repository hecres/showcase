# コメント対象範囲

## 1. コメント必須範囲

以下の要素には **必ず** XMLドキュメントコメントを付与すること。

| 対象 | 可視性条件 |
|------|-----------|
| クラス / インターフェース / 構造体 / 列挙体 | 全て（可視性問わず） |
| メソッド | 全て（可視性問わず） |
| プロパティ（式形式 `=>` / `get; set;` 含む） | public / protected |
| 定数 | public |
| フィールド | public |

> **注意**: メソッドは `private` / `internal` を含む **全アクセス修飾子** が対象。プロパティ・定数・フィールドのような可視性制限はメソッドには適用されない。

### 判定メモ

- `=>` または `get; set;` を持つ宣言はプロパティ扱い
- `operator` はコメント任意

## 2. コメント禁止範囲

以下の要素には **AIエージェントは新規コメントを付与してはならない**。

- 上記「必須範囲」に該当しない全ての要素

ただし、複雑な内容の場合は人間側でコメントを付与する場合がある。
そのため既存コメントは、内容が正しい限り削除しない（文体調整は可）。

## 3. 例外ルール

| 条件 | 扱い |
|------|------|
| 自動生成ファイル | コメント対象外 |
| `[Serializable]` クラスの public フィールド | コメント省略可 |
| `partial` クラス | 代表側（`ClassName.cs`）にのみ付与 |
| `operator` | コメント任意 |

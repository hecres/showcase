# 文体・語尾ルール

## 型（クラス / インターフェース / 構造体 / 列挙体）

- **常体言い切り**、句点なし
- 例:
  ```csharp
  /// <summary>
  /// ユニットの管理インターフェース
  /// </summary>
  ```

特殊パターン:

| パターン | 文言 |
|---------|------|
| 抽象クラス | 「〜クラスの基底」 |
| 自動生成クラス | 説明の前に「[自動生成ファイル]」を付記 |

## メソッド

- **敬体**「〜します。」、句点あり
- 例:
  ```csharp
  /// <summary>
  /// ダメージを計算します。
  /// </summary>
  ```

特殊パターン:

| パターン | 文言 |
|---------|------|
| コンストラクタ | 固定文言「コンストラクタ」（句点なし） |

## プロパティ / 定数 / フィールド

- **常体言い切り**、句点なし
- 例:
  ```csharp
  /// <summary>
  /// 最大HP
  /// </summary>
  ```

## よくある文体違反

以下は頻出の違反パターンである。

| 要素 | 正しい文体 | よくある誤り | 例 |
|------|-----------|------------|-----|
| メソッド summary | 敬体「〜します。」（句点あり） | 常体「〜する」（句点なし） | ✗「初期化する」→ ✓「初期化します。」 |
| 型 summary | 常体言い切り（句点なし） | 末尾に句点「。」が付いている | ✗「操作クラス。」→ ✓「操作クラス」 |
| プロパティ | 常体言い切り（句点なし） | 末尾に句点「。」が付いている | ✗「最大HP。」→ ✓「最大HP」 |

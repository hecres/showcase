# PR作成

プロジェクト共通のフォーマットに従ってGitHub PRを作成する。

## 参照

- [貢献タイプ分類](references/pr-contribution-types.md)
- [PR本文の視覚化指針](references/pr-visual-style.md)
- [PR本文テンプレート](templates/pr-body.md)

## 手順

- [ ] 1. 上記の参照ファイルをすべて読み込む
- [ ] 2. PRメッセージ（タイトル・概要・変更内容・備考すべて）は `main..HEAD` のコミット済み差分のみに言及する。差分に現れない事項——未コミット変更（作業ツリー差分）、会話文脈で触れただけの作業、別PR・別タスクとして切り出した対応、「本PRに含めず別途対応」等の除外事項——は確認・報告・備考記載のいずれも行わない
- [ ] 3. `git log main..HEAD --oneline` と `git diff main...HEAD --stat` でブランチの全変更を把握する
- [ ] 4. 貢献タイプを判定する（セッション内の文脈から判定できない場合はユーザーに確認する）
- [ ] 5. タイトルフォーマットに従ってPRタイトルを作成する
- [ ] 6. PR本文テンプレートに従って本文を作成する
- [ ] 7. セルフレビューを実施する
- [ ] 8. リモートへプッシュされていなければ `git push -u origin HEAD` でプッシュする
- [ ] 9. `gh pr create` でPRを作成し、PR URLを報告する

## タイトルフォーマット

```
[貢献アイコン] 対応内容を端的にまとめた一文
```

- 貢献アイコンは貢献タイプ分類に従って選択する（🤖 / 🤝 / 👤）
- 対応内容は変更全体の目的を端的に表す一文とする
- ベースブランチは明示指定がない限り `main` とする

## セルフレビュー

- [ ] 貢献タイプが正しく判定されているか
- [ ] タイトルが変更全体の目的を端的に表しているか
- [ ] 概要が「なぜ」を1〜2文で説明しており、変更内容と重複していないか
- [ ] 変更内容が表またはMermaid図で構造的に提示されているか（小規模変更を除く）
- [ ] 散文が複数文になる箇所で句点ごとに改行が入っているか（GFMでは `<br/>` 不要）
- [ ] 変更内容がブランチの変更を漏れなく反映しているか
- [ ] 全セクション（概要・変更内容・備考）が `main..HEAD` の差分内の事項のみに言及しているか（別途対応・除外事項・会話文脈のみの作業に触れていないか）
- [ ] ベースブランチが正しいか

# 型summaryの構文パターン

型（クラス / インターフェース / 構造体 / 列挙体）の `summary` は以下の構文に従う。
文体ルール（常体言い切り、句点なし）を前提として、型種別ごとに構文を定義する。

## 基本構文

```
<summary>[ドメイン説明]の[役割語][型種別]</summary>
```

## クラス

| パターン | 構文 | 例 |
|---------|------|-----|
| 通常 | `〜の[役割語]クラス` | `ユニットの管理クラス` |
| 抽象クラス / Base接尾辞 | `〜クラスの基底` | `型ラップ値クラスの基底` |
| 不変データ / Data接尾辞 | `〜の不変データクラス` | `属性の不変データクラス` |
| 可変データ | `〜の可変データクラス` | `進行状態の可変データクラス` |
| 拡張クラス | `〜の拡張クラス` | `Vector3の拡張クラス` |
| 自動生成 | `[自動生成ファイル] 〜` | `[自動生成ファイル] マスターデータ定義` |

## インターフェース

```
<summary>[ドメイン説明]の[役割語彙]インターフェース</summary>
```

## 構造体

| パターン | 構文 | 例 |
|---------|------|-----|
| 通常（値型） | `〜を表す構造体` | `ワールド座標を表す構造体` |
| 不変データ | `〜の不変データ構造体` | `経路座標の不変データ構造体` |

## 列挙体

| パターン | 構文 | 例 |
|---------|------|-----|
| タイプ名を含む場合 | `〜タイプの列挙体` | `所属タイプの列挙体` |
| タイプ名を含まない場合 | `〜の列挙体` | `原因の列挙体` |

**注意:** 「列挙型」は使用しない。常に「列挙体」と記述する。

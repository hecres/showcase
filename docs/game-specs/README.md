# ゲーム仕様

開発中ゲームの仕様書からの抜粋です。<br/>
仕様体系の根幹を示す3点と、記法を実例で確認できる代表仕様2点を掲載しています。

## 体系・原則

| ドキュメント | 内容 |
|---|---|
| [仕様策定原則](spec-principles.md) | ゲーム体験の核・リスクとリターン・テンポと爽快感など、仕様策定時に立ち返る判断基準 |
| [画面遷移図](spec-screen-flow.md) | アプリ全体の画面遷移フロー、クエスト攻略・カードデッキ編集の詳細フロー、UIレイヤー構造 |
| [仕様書フォーマット](spec-how-to-read.md) | 仕様書の記法ルール（`【】`パラメータ・暫定値・操作遷移一覧）と補足情報の種類（設計意図／技術補足／演出補足／仕様補足） |

## 非公開資料について

本ディレクトリは仕様書全体の一部抜粋です。<br/>
本書内のリンクで「（非公開資料）」と表記されている箇所は、実プロジェクトでは別の仕様書を参照していますが、公開範囲外のため[非公開資料の案内ページ](private-notice.md)へのリンクに置き換えています。
